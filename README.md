# АкваБлеск

Open-source проект для сайта клинингового сервиса с публичной частью, backend и админкой. Система включает клиентский лендинг, обработку заявок, CRM-модуль, управление отзывами, QR-кодами, фото `before/after`, настройками сайта и e-mail-уведомлениями.

## Стек

- HTML
- CSS
- JavaScript
- PowerShell backend
- JSON storage

## Что есть в проекте

- публичный сайт: hero, услуги, отзывы, FAQ, контакты, формы заявок, правовые страницы;
- backend API: приём заявок, авторизация, роли, reset password, загрузка файлов, экспорт в Excel, e-mail-уведомления;
- админка: заявки, фото `before/after`, отзывы, QR-коды, настройки сайта, личный кабинет;
- offline/dev fallback: админка может открываться локально даже без backend;
- связка фронтенда и админки: изменения настроек, отзывов, QR и фото подхватываются сайтом из общей модели данных.

## Локальный запуск

Запуск backend:

```powershell
powershell -ExecutionPolicy Bypass -File .\backend\server.ps1 -Port 8080
```

После запуска:

- сайт: `http://localhost:8080/`
- админка: `http://localhost:8080/admin/`
- личный кабинет: `http://localhost:8080/admin/security.html`
- настройки сайта: `http://localhost:8080/admin/site-settings.html`

Если backend не запущен, админка работает в локальном dev/offline режиме на mock-данных и `localStorage`.

## Dev-доступ

- администратор: `admin / admin`
- модератор: `moderator / moderator`

Это локальные dev-учётные записи. Для рабочей среды данные входа нужно заменить.

## Как работает админка

- `Заявки`: список обращений, создание, редактирование, смена статуса, подтверждение, удаление, Excel-экспорт;
- `Фото`: загрузка и управление блоками `before/after`;
- `Отзывы`: ссылки Яндекс/2ГИС, QR-коды, ручные отзывы, порядок и видимость;
- `Настройки`: телефон, адрес, график, MAX, карта, почта уведомлений;
- `Личный кабинет`: данные администратора и безопасность.

## Заявки и CRM

- формы сайта отправляют данные в backend;
- заявка сохраняется в хранилище и появляется в админке;
- доступны статусы, комментарии, дата/время, подтверждение выезда;
- владелец может скачать CRM-данные в `.xlsx`;
- в dev-режиме есть демо-набор заявок для проверки фильтров, таблицы и Excel-экспорта.

## Отзывы, Яндекс / 2ГИС и QR

- в админке настраиваются ссылки на Яндекс Карты / Яндекс Бизнес и 2ГИС;
- можно загружать официальные QR-коды;
- можно использовать ручные отзывы или официальный виджет/бейдж, если он доступен;
- фронтенд берёт эти данные из общей модели и показывает их в блоке отзывов.

## E-mail-уведомления

- рабочий e-mail владельца задаётся в настройках сайта;
- после новой заявки backend может отправить письмо владельцу;
- при включённой опции можно отправлять клиенту письмо-подтверждение;
- для отправки нужны SMTP-переменные окружения.

## Что можно менять без правки кода

Через админку:

- телефон и контакты;
- адрес и график;
- ссылку на MAX;
- карту и маршрут;
- ссылки Яндекс / 2ГИС;
- QR-коды;
- отзывы;
- фото `before/after`;
- e-mail для уведомлений;
- заявки и CRM-данные.

Через `assets/`:

- статические изображения;
- иконки;
- favicon и медиафайлы.

## Ограничения текущей версии

- backend использует JSON-хранилище, а не полноценную SQL-БД;
- live-интеграции внешних отзывов зависят от доступных официальных механизмов;
- SMTP работает только при корректной настройке окружения;
- GitHub Pages подходит только для статической части, серверные функции требуют backend.
